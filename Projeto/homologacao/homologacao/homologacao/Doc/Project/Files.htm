Arquivos do sistema


Durante sua utilização, a aplicação utiliza-se de vários arquivos que têm seus dados consultados e modificados de acordo com a necessidade. O projeto GerProjeto - Gerenciamento de Atividades utiliza-se dos seguintes arquivos e diretórios:

Arquivo Descrição
web.configPrincipal arquivo de configuração das aplicações ASP.NET, é um documento XML que contém diversas informações dos módulos, como qual módulo que devem ser carregados, Configurações de Segurança e Linguagem de compilação
global.asaxO arquivo Global.asax, também conhecido como arquivo do aplicativo ASP.NET, é um arquivo que contém código para responder a eventos no nível do aplicativo e nível de sessão-criados pelo ASP.NET ou pelo Módulos HTTP opcional.
Diretório App_GlobalResourcesDiretório que contem os arquivos de resources de projeto, solução e bancos de dados
Diretório DatabasesDiretório onde serão gravados os arquivos de criação do banco de dados SQL e XML de definição de banco.
Diretório PagesDiretório padrão onde serão armazenadas as páginas da aplicação.

web.config

Geralmente, o arquivo que possui o nome web.config é utilizado para armazenar parâmetros de configuração que são comuns a qualquer aplicação .Net, por exemplo, receber a sequência de comandos para o acesso a um banco de dados (string de conexão). Um dos parâmetros de uma string de conexão é o nome do servidor de dados que pode ser alterado de acordo com a necessidade. Quando o nome do servidor é alterado, essa alteração é gravada no arquivo web.config e a conexão com o novo servidor apresentado será então proporcionada à aplicação.
